# CS-210-Porfolio
## Project Reflection

### 1. Summarize the project and what problem it was solving. 
For this project, I developed a usable C++ application designed to solve simple data tracking problems. I focused on building a clean, user-friendly program that helps input data, handles math calculations efficiently, and displays clear reports for the user.  
### 2. What did you do particularly well?
I believe that I did really well with the code design and the input validation. I spent my time making sure that my object-oriented code was clean and that my input loops were running. In this particular senario i wanted to make sure that if a user accidentally types an invalid character or a negative number, the program would manage it with a clear error message instead of just crashing.

### 3. Where could you enhance your code? How would these improvements make your code more efficient, secure, and so on?
I would enhance the application by adding an advanced file exception tool to help protect the program from missing, empty, or corrupted data files. I would also enhance memory usage and speed processing by upgrading my search loop to use advanced data organization, like a hash map, which would keep the program running at a fast pace. 

### 4. Which pieces of the code did you find most challenging to write, and how did you overcome this? What tools or resources are you adding to your support network?
One of the most challenging parts for me has been managing file input/output workflows when constantly trying to organize the data into clean, perfectly aligned tables. I was able to work through this by testing smaller pieces of the functions individually and looking into the official C++ libraries. As I move forward, I have decided to officially add language documentation, developer, and structured peer code reviews to the support network.

### 5. What skills from this project will be particularly transferable to other projects or course work? 
Learning how to separate logic cleanly into independent classes and distinct functions is a skill that will strongly help me as I move into other languages like Java and Python. 

### 6. How did you make this program maintainable, readable, and adaptable?
I was able to make sure this program was highly maintainable by following clean coding standards. For instance, I used detailed variable names that made sense, kept my work perfectly uniform, and organized it into separate header (.h) and (.cpp) files. Also, I made sure to add clear comments explaining what each class and function does, so that if another person/developer (or my future self) needs to make changes or expand the code, this way will be easier to read and adjust.
